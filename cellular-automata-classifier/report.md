# Automated Classification of Emergent Patterns in Conway's Game of Life using Unsupervised Machine Learning

## Abstract

This report details a computational project designed to automatically classify emergent patterns in John Conway's Game of Life. The system utilizes unsupervised machine learning to categorize stable structures that arise from random initial conditions. A dataset was generated by running numerous simulations to stabilization. Key quantitative features—such as size, density, and temporal dynamics—were extracted from the resulting patterns. A K-Means clustering algorithm was trained on this feature data to group patterns into distinct categories. The resulting model, integrated into an interactive web application, successfully identifies and labels patterns as "Still Lifes" of varying sizes or "Oscillators" in real-time, providing a robust framework for the analysis of emergent phenomena in cellular automata.

## 1. Introduction

Conway's Game of Life is a zero-player game, meaning its evolution is determined by its initial state, requiring no further input. It is a classic example of a cellular automaton where complex, self-organizing patterns emerge from a simple set of rules. These emergent patterns are typically classified by their temporal behavior, such as still lifes (static), oscillators (periodic), and spaceships (translating). Manual classification is trivial for experts but impractical for large-scale, automated analysis.

This project addresses the challenge of automated classification. The primary objective was to develop a system that can simulate the Game of Life and, upon the simulation reaching a stable state, automatically identify and classify all emergent patterns using an unsupervised machine learning model.

## 2. Methodology

The methodology is composed of three primary stages: the mathematical formulation of the simulation environment, the engineering of features to describe the patterns, and the development of the unsupervised clustering model.

### 2.1 Conway's Game of Life: Mathematical Framework

The Game of Life is played on a two-dimensional grid of cells, where each cell can be in one of two states: alive (1) or dead (0). The grid evolves in discrete time steps.

Let $G^t$ be the matrix representing the grid at time step $t$, where an element $G_{i,j}^t \in \{0, 1\}$ is the state of the cell at row $i$ and column $j$. The state of a cell in the next generation, $G_{i,j}^{t+1}$, is determined by the states of its eight neighbors in a Moore neighborhood.

The number of live neighbors for a cell $(i,j)$ at time $t$ is given by the sum:
$$N_{i,j}^t = \left(\sum_{k=-1}^{1} \sum_{l=-1}^{1} G_{i+k, j+l}^t\right) - G_{i,j}^t$$

The grid is typically treated as a torus, meaning the boundaries wrap around. This is implemented efficiently using a 2D convolution with a specific kernel:
$$K = \begin{pmatrix} 1 & 1 & 1 \\ 1 & 0 & 1 \\ 1 & 1 & 1 \end{pmatrix}$$

The neighbor count matrix $N^t$ is then $N^t = \text{convolve2d}(G^t, K, \text{mode='same'}, \text{boundary='wrap'})$.

The update rules for the state transition $G_{i,j}^t \to G_{i,j}^{t+1}$ are:

1. **Survival:** A live cell ($G_{i,j}^t = 1$) with 2 or 3 live neighbors survives.
   $$(G_{i,j}^t = 1) \land (N_{i,j}^t \in \{2, 3\}) \implies G_{i,j}^{t+1} = 1$$

2. **Birth:** A dead cell ($G_{i,j}^t = 0$) with exactly 3 live neighbors becomes a live cell.
   $$(G_{i,j}^t = 0) \land (N_{i,j}^t = 3) \implies G_{i,j}^{t+1} = 1$$

3. **Death:** All other cells die or remain dead (due to underpopulation or overpopulation).
   $$G_{i,j}^{t+1} = 0 \text{, otherwise}$$

### 2.2 Feature Engineering

To classify patterns, each distinct, connected component within a stable cycle is described by a set of five quantitative features:

1. **Number of Live Cells ($n_{live}$):** The total count of live cells in the pattern's first state. For a pattern subgrid $P$, $n_{live} = \sum_{i,j} P_{i,j}$.

2. **Bounding Box Area ($A_{bbox}$):** The area of the smallest rectangle enclosing the pattern. $A_{bbox} = (\text{width}) \times (\text{height})$.

3. **Density ($\rho$):** The ratio of live cells to the bounding box area. $\rho = n_{live} / A_{bbox}$.

4. **Periodicity ($P$):** The number of unique grid states in a stable cycle. For a still life, $P=1$. For an oscillator, $P>1$.

5. **Center of Mass Change ($\Delta_{CoM}$):** The temporal variation in the pattern's center of mass over one period. For a pattern cycle $\{P_1, P_2, \dots, P_P\}$, the center of mass for each state $P_t$ is calculated. $\Delta_{CoM}$ is the sum of the standard deviations of the center of mass coordinates over the cycle. For all still lifes and non-translating oscillators, this value is 0.

### 2.3 Unsupervised Clustering Model: K-Means

K-Means clustering was chosen to partition the pattern data into groups. The algorithm aims to partition $n$ observations into $k$ clusters in which each observation belongs to the cluster with the nearest mean (cluster centroid).

First, the feature data is standardized using a Z-score transformation to ensure each feature contributes equally to the distance calculation. For a feature value $x$ with mean $\mu$ and standard deviation $\sigma$, the scaled value $z$ is:
$$z = \frac{x - \mu}{\sigma}$$

The K-Means algorithm minimizes the within-cluster sum of squares (WCSS), also known as inertia. The objective function is:
$J = \sum_{j=1}^{k} \sum_{\mathbf{x}_i \in C_j} || \mathbf{x}_i - \mathbf{\mu}_j ||^2$

where $k$ is the number of clusters, $C_j$ is the $j$-th cluster, $\mathbf{x}_i$ is a data point (feature vector) in cluster $C_j$, and $\mathbf{\mu}_j$ is the centroid of cluster $C_j$.

The algorithm proceeds as follows:

1. **Initialization:** $k$ initial centroids are chosen.
2. **Assignment Step:** Each data point is assigned to its nearest centroid based on Euclidean distance.
3. **Update Step:** The centroids are re-calculated as the mean of all data points assigned to that cluster.

Steps 2 and 3 are repeated until the cluster assignments no longer change.

## 3. Model Training and Interpretation

### 3.1 Data Generation

A dataset of 20,000 patterns was generated by running simulations on 30x30 grids with random initial states. Each simulation ran until it stabilized into a static or oscillating cycle, and features were extracted from every resulting pattern.

### 3.2 Determining the Optimal Number of Clusters ($k$)

The Elbow Method was employed to find the optimal value for $k$. By plotting the WCSS for a range of $k$ values, an "elbow" was observed at $k=4$, indicating the point of diminishing returns for adding more clusters. This suggested that the emergent patterns in the dataset naturally fall into four distinct categories.

### 3.3 Cluster Interpretation

After training the K-Means model with $k=4$, the mean feature values for each cluster were analyzed to assign a meaningful label:

- **Cluster 0: Large Still Life:** Characterized by high `n_live_cells` and `bounding_box_area`. Crucially, `periodicity` is 1 and `center_of_mass_change` is 0, confirming a static nature.

- **Cluster 1: Small Still Life:** Characterized by very low `n_live_cells` and `bounding_box_area`, with `periodicity` of 1 and no change in center of mass. This is the most common category.

- **Cluster 2: Oscillator:** The defining characteristic of this cluster is `periodicity` > 1. These patterns exhibit periodic behavior.

- **Cluster 3: Medium Still Life:** Represents patterns with `n_live_cells` and `bounding_box_area` between the small and large still life clusters, while still having a `periodicity` of 1.

## 4. System Implementation and Usage

The complete system is deployed as an interactive web application using Streamlit. It allows users to run simulations and see the machine learning model classify patterns in real-time.

### Basic Usage Steps:

1. **Configure Simulation:** Use the sidebar to select a starting pattern (e.g., "Random", "Glider", "Pulsar") and set the grid size.

2. **Run Simulation:** Click the "Run to Stabilization" button to start the simulation. The application will evolve the grid generation by generation until a stable cycle is detected. The final grid state is displayed.

3. **Review Generations:** After the simulation completes, a slider appears. The user can move the slider to review the grid at any generation, from the initial state to the final one.

4. **View Classifications:** The application automatically identifies all distinct patterns in the final stable state, draws bounding boxes around them, and labels them with their classification (e.g., "Small Still Life", "Oscillator") as determined by the trained K-Means model. The user can toggle the visibility of these boxes and labels.

## 5. Conclusion

This project successfully demonstrates the application of unsupervised machine learning to the complex domain of cellular automata. By systematically defining and extracting key features, a K-Means clustering model was able to discern meaningful, high-level categories of emergent patterns from low-level grid data. The resulting interactive application serves as both a powerful analytical tool and an effective demonstration of how machine learning can be used to discover structure in complex systems.

Future work could involve expanding the feature set to differentiate between types of oscillators or to identify "spaceships"—patterns that translate across the grid. Additionally, other clustering algorithms or supervised models could be explored for more granular classification.
